# Testing Plan

本文档记录 Personal Computer Knowledge Base 的测试计划。

当前 Stage 0 只创建文档和目录，不实现可运行程序，因此本阶段测试重点是文件结构和设计一致性检查。

## 1. Stage 0 检查

需要确认：

- `README.md` 已创建。
- `requirements.md` 已创建。
- `database_design.md` 已创建。
- `development_plan.md` 已创建。
- `testing_plan.md` 已创建。
- `changelog.md` 已创建。
- `.gitignore` 已创建。
- 基础目录已创建。
- `AGENTS.md` 未被修改。
- `ui_design.md` 未被修改。
- `assets/ui_preview.png` 未被覆盖。

## 2. 后续启动检查

在实现 PySide6 窗口后，需要确认：

- 应用可以正常启动。
- 顶部栏显示应用名、搜索框、新建卡片、导入和设置入口。
- 页面为三列布局。
- 左侧包含分类和底部草稿。
- 左侧不包含收藏。
- 左侧不包含常用标签。
- 分类中包含“论文笔记”。

## 3. 后续数据库检查

在实现 SQLite 数据层后，需要确认：

- 数据库使用 `cards` 作为核心表。
- 草稿通过 `is_draft = 1` 区分。
- 正式卡片通过 `is_draft = 0` 区分。
- 没有创建 `todos` 表。
- 没有创建 `favorites` 表。
- 测试不删除真实用户数据。

## 4. 后续知识卡片检查

需要确认：

- 可以创建正式知识卡片。
- 可以查看正式知识卡片。
- 可以编辑正式知识卡片。
- 删除正式知识卡片前有确认。
- 重启应用后数据仍然存在。

## 5. 后续草稿检查

需要确认：

- 可以在左侧底部快速创建草稿标题。
- 草稿只显示在左侧草稿区。
- 草稿不出现在中间正式卡片列表。
- 点击草稿标题后，右侧直接进入编辑模式。
- 保存为正式卡片后，`is_draft` 从 `1` 改为 `0`。
- 保存后的卡片从草稿区消失，并进入正式卡片列表。
- 删除草稿前有确认。

## 6. 后续 Markdown 检查

需要确认 Markdown 预览至少能显示：

- 标题
- 段落
- 有序列表
- 无序列表
- 代码块
- 行内代码
- 链接
- 引用
- 简单表格

代码块需要使用清晰的等宽字体显示。

## 7. 后续搜索检查

需要确认搜索范围包括：

- 标题
- 摘要
- 标签
- 关键词
- 正文
- 分类
- 使用场景

第一版搜索只需要覆盖正式知识卡片。草稿可以暂不参与搜索。
