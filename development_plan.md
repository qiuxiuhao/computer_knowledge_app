# Development Plan

本文档记录项目开发阶段计划。

## Stage 0：项目文档和基础目录

目标：

- 创建 README 和需求文档。
- 创建数据库设计文档。
- 创建开发计划、测试计划和更新日志。
- 创建基础目录结构。
- 不实现业务代码。

本阶段不做：

- PySide6 窗口
- SQLite 数据库
- 真实搜索
- Markdown 预览
- 收藏功能
- 独立待补充页面
- 常用标签侧栏

## Stage 1：最小 PySide6 窗口

目标：

- 创建可启动的 PySide6 主窗口。
- 搭建三列布局。
- 使用模拟数据展示分类、草稿、卡片列表和右侧内容。

注意：

- 不连接数据库。
- 不实现真实搜索。
- 不实现真实 Markdown 渲染。
- 不添加收藏、独立待补充页面或常用标签侧栏。

## Stage 2：SQLite 数据层

目标：

- 初始化本地 SQLite 数据库。
- 创建 `cards` 核心表。
- 实现基础数据访问方法。

注意：

- 草稿使用 `cards.is_draft = 1`。
- 正式卡片使用 `cards.is_draft = 0`。
- 不创建 `todos` 表。
- 不创建 `favorites` 表。

## Stage 3：知识卡片 CRUD

目标：

- 新建正式知识卡片。
- 查看正式知识卡片。
- 编辑正式知识卡片。
- 删除正式知识卡片，并在删除前确认。

## Stage 4：Markdown 预览和编辑模式

目标：

- 正式卡片默认显示 Markdown 预览。
- 点击编辑后显示 Markdown 原文。
- 保存后回到预览模式。
- 取消编辑时放弃当前修改。

## Stage 5：搜索

目标：

- 支持关键词搜索正式知识卡片。
- 搜索范围包括标题、摘要、标签、关键词、正文、分类和使用场景。
- 第一版可使用 SQLite `LIKE`。

## Stage 6：草稿区

目标：

- 左侧底部显示草稿区。
- 支持快速输入标题创建草稿。
- 草稿显示为短标题列表。
- 点击草稿后，右侧直接进入编辑模式。
- 草稿保存为正式卡片后，从草稿区移除。

## Stage 7：体验优化

目标：

- 优化空状态提示。
- 优化删除确认。
- 优化搜索结果排序。
- 优化代码块显示效果。
- 优化三列布局细节。

## Stage 8：macOS 打包

目标：

- 在核心功能稳定后，考虑打包为 macOS 应用。
- 打包前确认数据目录、备份目录和用户隐私数据不会被错误覆盖或提交。
