# Requirements

本文档记录 Personal Computer Knowledge Base 第一版的需求范围。

## 1. 产品目标

帮助用户快速记录并找回个人计算机学习过程中的知识和经验，包括：

- 计算机术语
- Python 学习笔记
- 深度学习基础
- PyTorch 示例代码
- 终端命令
- macOS 使用技巧
- AutoDL 环境配置
- conda / pip / tmux / git 使用经验
- 报错信息和解决方案
- 项目实践经验
- 论文笔记
- 以后想补充的知识卡片草稿标题

## 2. 技术栈

项目技术栈保持为：

- Python
- PySide6
- SQLite
- Markdown

不得在未确认的情况下改成 Flask、Django、FastAPI、Electron、Web 前端或云端应用。

## 3. 第一版页面范围

第一版只保留一个核心页面：

```text
知识库主页面
```

页面布局：

```text
顶部：应用名 / 当前页面 / 搜索 / 新建卡片 / 导入 / 设置入口
左侧：分类 + 底部草稿
中间：正式知识卡片列表
右侧：卡片预览 / 编辑
```

第一版不包含：

- 独立待补充页面
- 收藏页面或收藏按钮
- 常用标签侧栏
- AI 问答页面
- 云同步页面
- 登录页面
- 复杂任务管理页面

## 4. 知识卡片需求

知识卡片是一条围绕某个主题的完整记录，可以包含：

- 概念解释
- 示例代码
- 终端命令
- 报错信息
- 解决步骤
- 注意事项
- 项目经验
- 相关链接
- 论文阅读笔记

知识卡片不需要严格区分为“概念卡”“代码卡”“报错卡”等类型。

## 5. 草稿需求

草稿放在左侧栏底部。

草稿只用于快速记下以后想补充的知识卡片标题。

草稿规则：

- 草稿是 `is_draft = 1` 的知识卡片。
- 草稿不是待办事项。
- 草稿没有优先级。
- 草稿没有完成状态。
- 草稿不使用独立详情页。
- 点击草稿标题后，右侧直接进入编辑模式。
- 保存为正式卡片后，`is_draft` 改为 `0`。

## 6. 字段需求

第一版知识卡片字段：

| 字段 | 含义 |
|---|---|
| `id` | 唯一卡片 ID |
| `title` | 标题 |
| `scenario` | 使用场景 |
| `category` | 分类 |
| `tags` | 标签 |
| `summary` | 一句话总结 |
| `content` | Markdown 原文内容 |
| `keywords` | 额外搜索关键词 |
| `source` | 来源链接或来源说明 |
| `is_draft` | 是否为草稿 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

第一版不要添加：

- `status`
- `mastery_status`
- `learning_status`
- `difficulty_level`
- `review_status`
- `is_favorite`
- `favorite_status`

## 7. 分类需求

第一版分类必须包含：

- 全部
- Python
- 深度学习
- PyTorch
- AutoDL
- Mac
- Linux
- Conda
- Git
- 常用命令
- 报错解决
- 项目经验
- 论文笔记

其中“论文笔记”用于记录论文阅读、模型分析和方法总结。

## 8. 搜索需求

第一版后续计划支持普通关键词搜索，范围包括：

- 标题
- 摘要
- 标签
- 关键词
- 正文
- 分类
- 使用场景

当前 Stage 0 只记录需求，不实现真实搜索。

## 9. Markdown 需求

知识卡片正文计划以 Markdown 原文保存。

后续正式卡片默认显示 Markdown 预览，点击编辑后显示 Markdown 原文。

当前 Stage 0 只记录需求，不实现 Markdown 预览。
